# Specs
Фичи
1. Депозит денег пользователем
2. Перевод денег с одного аккаунта на другой
3. Изменение имени пользователя

Требования к API:
Данные администратора: admin admin
В приложении используется базовая аунтефикация

По ролям: Админ:
1. Админ может создавать юзеров
2. Админ может видеть список юзеров

По ролям: Юзер:
1. Юзер может логиниться и получать токен
2. Авторизованный юзер может видеть свои счета
3. Авторизованный юзер может создавать новый счет
4. Авторизованный юзер может пополнять свой счет
5. Авторизованный юзер может пересылать деньги с одного счета на другой
6. Авторизованный юзер может видеть транзакции своих счетов
7. Авторизованный юзер может изменить имя в профиле

# exercise
Вам необходимо спроектировать тест кейсы для API из итерации 2

## Изменение имени пользователя
### Требования
Имя в профиле (name):
— Два слова, состоящее из букв, разделенные пробелом

### Кейсы
- Позитив: Два слова по одному символу (латиница) "a B"
- Негатив: Пробел " "
- Негатив: Пустая строка ""
- Негатив: Одно слово "a"
- Негатив: Два слова, одно составное через дефис "Петрова Наталья-Анастасия"
- Негатив: Спец символ и слово ". B"
- Негатив: Передать невалидный тип данных (число) 1
- Негатив: Существующий аккаунт, передать null

## Депозит денег пользователем
### Требования
Депозит (Deposit):
— Максимальная сумма: 5000
— Сумма должна быть положительной
— Нельзя делать депозит в чужой аккаунт или несуществующий

### Кейсы
- Позитив: Существующий аккаунт, 1
- Позитив: Существующий аккаунт, 5000
- Негатив: Не существующий аккаунт, 1
- Негатив: Чужой аккаунт, 1
- Негатив: Существующий аккаунт, 0
- Негатив: Существующий аккаунт, 0.1
- Негатив: Существующий аккаунт, -1
- Негатив: Существующий аккаунт, 5001
- Негатив: Существующий аккаунт, передать невалидный тип данных - строка
- Негатив: Существующий аккаунт, передать null

## Перевод денег с одного аккаунта на другой
### Требования
Перевод (Transfer):
— Максимальная сумма: 10000
— Сумма должна быть положительной и не превышать баланс отправителя
— Можно переводить между своими аккаунтами и чужими

### Кейсы
- Позитив: Перевод между своими существующими аккаунтами, баланс 10000, перевод 1
- Позитив: Перевод между своими существующими аккаунтами, баланс 10000, перевод 10000
- Позитив: Перевод со своего аккаунт на чужой существующий аккаунт, баланс 10000, перевод 1
- Позитив: Перевод со своего аккаунт на чужой существующий аккаунт, баланс 10000, перевод 10000
- Негатив: Перевод между своими существующими аккаунтами, баланс 10000, перевод 0
- Негатив: Перевод между своими существующими аккаунтами, баланс 10001, перевод 10001
- Негатив: Перевод со своего аккаунт на чужой существующий аккаунт, баланс 10000, перевод 0
- Негатив: Перевод со своего аккаунт на чужой существующий аккаунт, баланс 10000, перевод 10001
- Негатив: Перевод с несуществующего аккаунта на свой существующий аккаунт, баланс 10000, перевод 1
- Негатив: Перевод с существующего аккаунта на несуществующий аккаунт, баланс 10000, перевод 1
- Негатив: Перевод с чужого аккаунта на свой аккаунт, баланс 10000, перевод 1
- Негатив: Перевод между своими существующими аккаунтами, баланс 10000, перевод 100000
- Негатив: Перевод между своими существующими аккаунтами, баланс 10000, перевод -1
- Негатив: Перевод между своими существующими аккаунтами, баланс 10000, перевод 0.1
- Негатив: Перевод между своими существующими аккаунтами, передать в senderAccountId null
- Негатив: Перевод между своими существующими аккаунтами, передать в receiverAccountId null
- Негатив: Перевод между своими существующими аккаунтами, передать в amount null
- Негатив: Перевод между своими существующими аккаунтами, передать в senderAccountId невалидный тип данных - строка
- Негатив: Перевод между своими существующими аккаунтами, передать в receiverAccountId невалидный тип данных - строка
- Негатив: Перевод между своими существующими аккаунтами, передать в amount невалидный тип данных - строка
